Repositories define the contract for data access. They are implemented in the data layer and used by use cases in the domain layer.

## CartRepository

Manages shopping cart data persistence and retrieval.

**Source:** `cart-component/src/commonMain/kotlin/com/denisbrandi/androidrealca/cart/domain/repository/CartRepository.kt:6`

```kotlin
internal interface CartRepository {
    fun updateCartItem(userId: String, cartItem: CartItem)
    fun observeCart(userId: String): Flow<Cart>
    fun getCart(userId: String): Cart
}
```

### updateCartItem

Updates or adds a cart item for a specific user.

<ParamField path="userId" type="String" required>
  The ID of the user
</ParamField>

<ParamField path="cartItem" type="CartItem" required>
  The cart item to update or add
</ParamField>

**Returns:** `Unit`

---

### observeCart

Observes cart changes for a specific user.

<ParamField path="userId" type="String" required>
  The ID of the user
</ParamField>

**Returns:** `Flow<Cart>` - A flow emitting cart updates

<ResponseField name="Flow<Cart>" type="Flow<Cart>">
  Reactive stream of cart state changes
</ResponseField>

---

### getCart

Retrieves the current cart snapshot for a user.

<ParamField path="userId" type="String" required>
  The ID of the user
</ParamField>

**Returns:** `Cart` - The current cart state

<ResponseField name="Cart" type="Cart">
  Current cart with all items
</ResponseField>

## UserRepository

Manages user authentication and user data.

**Source:** `user-component/src/commonMain/kotlin/com/denisbrandi/androidrealca/user/domain/repository/UserRepository.kt:6`

```kotlin
internal interface UserRepository {
    suspend fun login(loginRequest: LoginRequest): Answer<Unit, LoginError>
    fun getUser(): User
    fun isLoggedIn(): Boolean
}
```

### login

Authenticates a user with the provided credentials.

<ParamField path="loginRequest" type="LoginRequest" required>
  Login credentials (email and password)
</ParamField>

**Returns:** `Answer<Unit, LoginError>` - Success or specific login error

<ResponseField name="Success" type="Unit">
  Authentication successful
</ResponseField>

<ResponseField name="Error" type="LoginError">
  One of: InvalidEmail, InvalidPassword, GenericError, IncorrectCredentials
</ResponseField>

---

### getUser

Retrieves the currently authenticated user.

**Returns:** `User` - The current user

<ResponseField name="User" type="User">
  Current authenticated user data
</ResponseField>

---

### isLoggedIn

Checks if a user is currently authenticated.

**Returns:** `Boolean` - True if user is logged in

<ResponseField name="Boolean" type="Boolean">
  Authentication status
</ResponseField>

---
