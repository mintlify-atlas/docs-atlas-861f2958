title: Repository Implementations
description: Data layer repository implementations that bridge domain logic with data sources

Repository implementations in the data layer provide concrete implementations of domain repository interfaces. They handle data operations, coordinate between local cache and remote APIs, and manage data transformations.

## RealCartRepository

Implements cart operations with local caching and reactive state management.

**Location:** `cart-component/src/commonMain/kotlin/com/denisbrandi/androidrealca/cart/data/repository/RealCartRepository.kt`

### Constructor

<ParamField path="cacheProvider" type="CacheProvider" required>
  Cache provider for persisting cart data
</ParamField>

### Methods

#### updateCartItem

Updates or removes a cart item for a specific user.

<ParamField path="userId" type="String" required>
  The user ID whose cart to update
</ParamField>

<ParamField path="cartItem" type="CartItem" required>
  The cart item to update. If quantity is 0, the item is removed
</ParamField>

```kotlin
fun updateCartItem(userId: String, cartItem: CartItem)
```

#### observeCart

Observes cart changes reactively for a specific user.

<ParamField path="userId" type="String" required>
  The user ID whose cart to observe
</ParamField>

<ResponseField name="return" type="Flow<Cart>">
  A Flow emitting cart updates
</ResponseField>

```kotlin
fun observeCart(userId: String): Flow<Cart>
```

#### getCart

Retrieves the current cart state for a user.

<ParamField path="userId" type="String" required>
  The user ID whose cart to retrieve
</ParamField>

<ResponseField name="return" type="Cart">
  The current cart state
</ResponseField>

```kotlin
fun getCart(userId: String): Cart
```
