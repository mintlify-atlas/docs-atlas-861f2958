## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the project from GitHub:
    
    ```bash
    git clone https://github.com/DenisBronx/Real-Clean-Architecture-In-Android.git
    cd Real-Clean-Architecture-In-Android
    ```
  </Step>
  
  <Step title="Open in Android Studio">
    1. Launch Android Studio
    2. Select **File > Open**
    3. Navigate to the cloned directory
    4. Click **OK**
    
    Android Studio will automatically sync Gradle and download dependencies.
  </Step>
  
  <Step title="Wait for Gradle Sync">
    The first sync may take a few minutes as Gradle downloads all dependencies. Watch the progress in the bottom status bar.
    
    <Note>
      If you encounter sync errors, try **File > Invalidate Caches > Invalidate and Restart**
    </Note>
  </Step>
  
  <Step title="Build the Project">
    Build the project to ensure everything is set up correctly:
    
    ```bash
    ./gradlew build
    ```
    
    Or use Android Studio's **Build > Make Project** menu.
  </Step>
  
  <Step title="Run the App">
    1. Connect an Android device or start an emulator
    2. Select the **app** run configuration
    3. Click the **Run** button (green triangle)
    
    The app should launch on your device/emulator.
  </Step>
</Steps>

## Project Structure

Once opened, you'll see the following module structure:

```bash
Real-Clean-Architecture-In-Android/
├── app/                          # Main application module
├── foundations/                  # Core utilities and Answer type
├── cache/                        # Key-value storage
├── httpclient/                   # Network client
├── designsystem/                 # Reusable UI components
│
├── user-component/               # User domain & data
├── product-component/            # Product domain & data
├── cart-component/               # Cart domain & data
├── wishlist-component/           # Wishlist domain & data
├── money-component/              # Money domain object
│
├── onboarding-ui/                # Login screen
├── plp-ui/                       # Product list screen
├── cart-ui/                      # Cart screen
├── wishlist-ui/                  # Wishlist screen
├── main-ui/                      # Main navigation
└── money-ui/                     # Price display components
```

<Note>
  **Component modules** (e.g., `user-component`) are pure Kotlin/KMP and compile much faster than UI modules.
</Note>

## Running Tests

The project includes comprehensive unit tests for all business logic.

### Run All Tests

```bash
./gradlew test
```

### Run Tests for Specific Module

```bash
./gradlew :user-component:test
./gradlew :cart-component:test
```

### Generate Code Coverage Report

The project uses Kover for code coverage:

```bash
./gradlew koverHtmlReport
```

Open the report at `build/reports/kover/html/index.html`

## Exploring the Code

### Key Examples to Study

<AccordionGroup>
  <Accordion title="Domain Layer - Use Cases">
    Check out `user-component/src/commonMain/kotlin/com/denisbrandi/androidrealca/user/domain/usecase/LoginUseCase.kt` to see pure business logic:
    
    ```kotlin
    internal class LoginUseCase(
        private val userRepository: UserRepository
    ) : Login {
        override suspend fun invoke(loginRequest: LoginRequest): Answer<Unit, LoginError> {
            return when {
                !Email(loginRequest.email).isValid() -> {
                    Answer.Error(LoginError.InvalidEmail)
                }
                !Password(loginRequest.password).isValid() -> {
                    Answer.Error(LoginError.InvalidPassword)
                }
                else -> {
                    return userRepository.login(loginRequest)
                }
            }
        }
    }
    ```
  </Accordion>
  
  <Accordion title="Data Layer - Repository">
    See how repositories implement domain interfaces in `user-component/src/commonMain/kotlin/com/denisbrandi/androidrealca/user/data/repository/RealUserRepository.kt`

  <Accordion title="Presentation Layer - ViewModel">

## Troubleshooting

<AccordionGroup>
  <Accordion title="Gradle Sync Failed">
    1. Check your internet connection
    2. Verify JDK 17 is installed and configured
    3. Try **File > Invalidate Caches > Invalidate and Restart**
    4. Delete `.gradle` folder and sync again
  </Accordion>
  
  <Accordion title="Build Errors">
    1. Ensure Android SDK 36 is installed via SDK Manager
    2. Check that `JAVA_HOME` points to JDK 17
    3. Run `./gradlew clean build --refresh-dependencies`
  </Accordion>
  
  <Accordion title="App Won't Run">
    1. Verify your device/emulator is running Android 7.0 (API 24) or higher
    2. Check that the device is connected: `adb devices`
    3. Try uninstalling and reinstalling the app
  </Accordion>
</AccordionGroup>
